# 人机协同指南 — GTM Agent Swarm

本文档规定了 GTM Agent 与人类 Reviewer 之间的协作流程与边界。所有 Agent 收到任务时，任务描述里会内嵌这份规则。

---

## 核心流程

```
Idea（想法）
  ↓ Promote（人类发起）
In Progress — Agent 制作初稿
  ↓ Agent 完成后
In Review — 等待人类审核
  ↓ Reviewer 通过
In Bank — 发布就绪
  ↓ 按时间调度
Published — 已发布
```

---

## Agent 行为规则

### 规则 1：只出草稿，不直接发布

Agent 完成内容后，**必须将任务状态改为 In Review**，等待人类审核。

**严禁行为：**
- 直接将内容发布到任何公开平台（Reddit、官网、X、YouTube 等）
- 在未经 Reviewer 确认的情况下触发任何对外分发动作

### 规则 2：Reviewer 通过才能进 Bank

- Reviewer 审核通过 → 状态改为 **In Bank**
- Reviewer 未通过 → Agent 根据反馈修改，重新提交 Review
- 循环直到 Reviewer 明确表示"通过"为止

### 规则 3：Bank 里的内容按时调度发布

| 情况 | 发布时间 |
|------|---------|
| 未指定发布时间 | 进入 In Bank 后 **1 小时** |
| 已指定发布时间 | 严格按照约定时间 |

**时区约定：所有时间统一使用 UTC+0。**
Reviewer 在约定发布时间时，请务必将本地时间转换为 UTC+0 再填写，避免时区错乱。

---

## Reviewer 职责

| 动作 | 说明 |
|------|------|
| 收到 In Review 的内容 | 仔细阅读，给出明确的通过或修改意见 |
| 通过 | 将状态改为 In Bank，可选填发布时间（UTC+0）|
| 不通过 | 留下具体反馈，Agent 会重新修改 |
| 指定发布时间 | 必须使用 UTC+0，并提醒自己当地时区对应的换算 |

---

## 常见时区换算参考

| 时区 | UTC 偏移 | 举例 |
|------|---------|------|
| 北京 / 上海（CST） | UTC+8 | 下午 3 点 = UTC 07:00 |
| 纽约（EST） | UTC-5 | 上午 9 点 = UTC 14:00 |
| 伦敦（GMT） | UTC+0 | 即 UTC，无需换算 |
| 洛杉矶（PST） | UTC-8 | 上午 9 点 = UTC 17:00 |

---

## 总结

> Agent 负责生产，人类负责把关。
> 内容未经 Reviewer 通过，绝不流出到公众面前。
> 所有时间以 UTC+0 为准。
