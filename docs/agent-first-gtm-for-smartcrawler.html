市场洞察

What it is · 差异化 · 与 Firecrawl/Apify 的位置差

这是什么 — 一句话

smartcrawler = 家居 DTC 跨境垂直情报 API，伪装成 "crawler"，实际是 Keepa for furniture。 数据维度：9 个家居品牌 × 46 个 Shopify/Vue/Nuxt 站 × 12 国 × 21 review platform（Trustpilot / Google Maps）+ Google Shopping。 输出预建模实体：Products / PriceHistory / Promotions / VOC reviews / CompetitorShare。

Agent-native 程度异常高：/.well-known/mcp.json · /.well-known/ai-plugin.json · /agents.json · /llms.txt · OpenAPI 3.1 · Swagger UI 全齐。 7 个 MCP tool 是 domain verb：search_competitor_products / voc_summary / competitor_landscape / ... — Firecrawl 给你 scrape()，他给你业务动作。

差异化 Wedge
垂直深度 ≠ 横向爬虫

Firecrawl / Browserless / ScrapingBee / Apify 卖通用原语 (scrape / crawl)。 smartcrawler 卖领域动词（competitor_landscape, voc_summary）。 形态对位 Keepa（Amazon），不是 Firecrawl。

家居 DTC only 7 MCP tools OpenAPI 3.1 llms.txt CN + EN bilingual
直接竞品（按位置）
  • Prisync / Competera / Altosight — 通用价格追踪 SaaS
  • Similarweb Shopper Intelligence — 更广的 e-commerce intel
  • Keepa — 同形态不同垂直（Amazon）
  • Firecrawl + DIY pipeline — 间接替代
  • ScrapeGraphAI — agent 友好但 horizontal
Agent-readiness 体检（同行业最佳）
  • ✅ /.well-known/mcp.json + /.well-known/ai-plugin.json
  • ✅ /agents.json + /llms.txt
  • ✅ OpenAPI 3.1 + Swagger UI public
  • ✅ MCP streamable HTTP, Bearer auth
  • ✅ REST via X-API-Key
  • 已经做到 GTM Swarm v3 想做的 90% — 是教科书级范本
最大的缺口
"Agent-native 满分，但没人知道它存在"

所有 discovery surface 全齐，但是 customer signal 完全空：无 logo / 无 case / 无 PR / 无 social。 产品已就绪 = Agent 来了能用，但 Agent 还没被告知它的存在。这是 GTM 任务全部的重心。

Persona — 谁会用 smartcrawler

Human + Agent 双买家 · Agent 是 unfair advantage

主买家 Human

"中型跨境家居 DTC 品牌的 Competitive Intelligence 分析师 / Pricing Manager / 品类 PM"

  • 公司: 家居 DTC · revenue $10-200M · 在 ≥ 3 国 / Shopify-based · 自有品牌
  • 身份: CI Analyst / Pricing / 品类 PM / 跨境运营总监
  • 当前痛: 维护 in-house 爬虫被 Shopify theme 改版搞挂 / Prisync 没覆盖独立站 / Excel 手动汇总
  • 预算: 数据 SaaS 月支出 $2K-$15K，可换
购买触发（4 选 1）
① 上线新国家，需要快速摸到本地竞品价位结构
② 黑五 / 双 11 前需要竞品 promo 监控 dashboard
③ 董事会要"我们 vs 头部 3 家"的 price-positioning 一页纸
④ 内部爬虫团队成本 / 跑数据 SLA 失控

真正决策者 Agent

"跨境品牌的 Pricing Agent / Competitor Watch Agent / Listing Optimizer"

  • 身份: 内部 LangGraph / Claude project agent，每天跑 competitor monitoring
  • 当前路径: Firecrawl + 自己写 parsing logic + 自己维护 schema → 慢、贵、脆
  • 切换动机: 一次 competitor_landscape("sofa", "us") 替代 46 次 scrape + 46 次 parser + 46 次 schema 维护
  • 限制: smartcrawler 只覆盖家居。Agent 如果跨垂直，必须保留 Firecrawl 作 fallback
Agent "选 smartcrawler" 的真实路径
① 在 MCP 配置里 add smartcrawler MCP server（已有 endpoint）
② Agent 跑 tools/list → 看到 7 个 domain verb
③ 看到 competitor_landscape 直接调 — 不需要写新代码
④ smartcrawler 在 agent 工具箱里成为 "家居场景的 default crawler"

抽四张卡 — Agent-First GTM for smartcrawler

产品已就绪 · GTM 任务全部在 Discovery + Distribution 两端
Card 01 / Discovery

发现 — 让 Agent 训练数据知道它

"Google SEO" → "LLM training data 收录 + MCP catalog 收录"。产品已经 agent-ready，需要做的只是让索引知道它。
五个动作
  1. 提交 5 大 MCP catalog：mcp.so / glama.ai / smithery.ai / modelcontextprotocol/servers / PulseMCP。1 天搞定，分发上去就有流量。
  2. 把 OpenAPI 提交到 apis.guru + RapidAPI。两个目录都是 LLM 训练数据高频源。
  3. 每个 MCP tool 的 description 字段重写，把"家居 DTC"明确写进去，让 Agent router 第一秒就 match domain。
  4. 建一个 vs/firecrawl / vs/apify 对比页 — agent router 训练数据高频抓 comparison。表达"我比 Firecrawl 更适合家居场景"是杀招。
  5. llms.txt 升级：把 7 个 MCP tool 的 example invocation 写齐，给 LLM 看的代码片段越具体越好。
本季度交付
5 catalog 全收录 · apis.guru / RapidAPI 收录 · vs/firecrawl 对比页 · 改良 llms.txt
KPI 每周 unique Agent runtime 探测 mcp.json ≥ 100
Card 02 / Traffic

引流 — 在 Agent 决策"用什么 crawler"时被选中

"广告投开发者" → "在 router 评估时赢 Firecrawl 一头"。决策线是domain match + cost + 数据 fidelity。
五个动作
  1. 放出免费 tier：每月 100 调用 free，无 signup wall。Agent 第一次接触零摩擦。
  2. 建 live demo MCP playground：/playground 让人/Agent 不登录就能跑 competitor_landscape("sofa", "us")，看实际 output。Demo 即销售。
  3. Fallback URL 引流：Firecrawl / Apify 用户在他们 dashboard 找不到家居垂直方案时，搜"furniture competitor data" 第一个看到 smartcrawler 的 cookbook。
  4. 跟 家居 DTC 工具生态（如 OrderProtection / Triple Whale / Shopify apps for furniture）打通 cross-link manifest，让他们的 agent 推荐 smartcrawler。
  5. One-shot install：claude mcp add smartcrawler -- npx -y @smartcrawler/mcp@latest。
本季度交付
公开 free tier · /playground live demo · @smartcrawler/mcp npm 包
KPI MCP install ≥ 50/周 · 30 天活跃调用率 ≥ 60%
Card 03 / Content

内容 — 每条都包含可调用 example

"爬虫如何工作"教程" → "垂直案例 + curl + agent dataflow"。家居 DTC 圈内 use case 越具体，圈外越不可替代。
五个动作
  1. 10 个 cookbook：track-competitor-sale-in-realtime / diff-product-listing-week-over-week / vc-summary-by-country / predict-stockout / price-position-deck ...
  2. 每个 cookbook 末尾自带 MCP install 命令 + 完整 curl + JSON output 示例。Agent 训练数据吸收即变 default。
  3. 每周 1 篇家居 DTC 行业报告："2026 Q2 美国 sofa 类目竞品价格变化趋势" — 数据来自 smartcrawler，自然背书。PR + LLM 训练数据 + SEO 三重收。
  4. Blog 文末 footer 强制带："Try it via MCP: npx @smartcrawler/mcp"。
  5. 开 每月一份 Agent 友好的 Industry Snapshot，让 Claude project / Cursor / 任何 Agent 直接调来给老板写月报。
本季度交付
10 cookbook · 12 周报告（每周一篇）· cookbook auto-sync 到 llms.txt
KPI cookbook 累计 GitHub stars ≥ 300 · 报告被引用率 ≥ 50/月
Card 04 / Distribution

分发 — 嵌入家居 DTC Agent 默认工具箱

"销售 BD" → "在所有家居 DTC Agent template 里默认带 smartcrawler"。垂直收窄 = unfair distribution。
五个动作
  1. 开源 3 个 reference agent template：shopify-competitor-watcher / furniture-pricing-agent / cross-border-promo-monitor。每个用 smartcrawler 当默认 data source。
  2. 跟 家居出海 community（小红书博主 / Shopify 出海公众号 / Furniture Today / Indistry Dive Retail）建立 inbound channel，把 cookbook + reference agent 推过去。
  3. 跟 Triple Whale / Lifetimely / Shopify analytics 类 app 谈整合，把 smartcrawler 作为 competitor 模块的数据 backend。Hub-and-spoke at industry layer。
  4. 埋点 tool_call_origin：知道哪个 agent runtime / 哪个 industry 调用最多，把客户行业分布 做成公开 dashboard 当 social proof。
  5. 跟我们集团内 voc.ai 集成：voc.ai 卖 Amazon review intelligence，smartcrawler 卖独立站 intel。Agent 装了 voc.ai 自动看到 smartcrawler 作为"独立站补充数据源"。集团内分发。

3 个 OSS reference agent · 1 个 Shopify app 整合 · voc.ai cross-link 上线
